# Indie Hackers Research Guide

## Purpose

Indie Hackers (indiehackers.com) is a community of bootstrapped entrepreneurs sharing
revenue milestones, business lessons, and monetization strategies. It's invaluable for
understanding what revenue levels are realistic for small-team games and which
approaches actually work.

Since Indie Hackers has no public API and uses JavaScript rendering, research is done
via **WebSearch** using site-specific queries.

## WebSearch Queries to Run

Run 3–5 of these queries based on the target game genre. Replace `[GENRE]` with
relevant terms (e.g., "mobile", "puzzle", "cozy", "steam", "idle").

### Revenue and success stories
```
site:indiehackers.com game revenue 2024
site:indiehackers.com indie game $1000 MRR
site:indiehackers.com "made $" game solo developer
site:indiehackers.com steam game launch revenue
site:indiehackers.com mobile game app store revenue
```

## What to Extract from Results

For each relevant result, capture:

1. **Revenue benchmark**: What is this type of game earning? ($100/mo? $5k/mo? $50k launch?)
2. **Monetization model**: Premium, freemium, subscription, IAP, DLC?
3. **Platform breakdown**: Mobile vs Steam vs both? Revenue split?
4. **Time to revenue**: How long from start to first dollar?
5. **Key marketing channel**: Where did early customers come from?
6. **Team size**: Solo, 2-person, small team?

## Common Patterns Found on Indie Hackers (Games)

### What works for small teams

- **Premium one-time purchase** on Steam: Lower volume, higher intent buyers, no
  ongoing support burden. Common for solo devs.
- **Free + optional IAP** on mobile: Higher download volume, monetize a small % of
  engaged players. Requires retention mechanics.
- **Premium mobile app** ($2.99–$9.99): Niche but sustainable for distinctive games
  without live service requirements.
- **Subscription add-ons**: Rare for games, but works for games with content libraries
  (level packs, skins) where players want ongoing content.

### Revenue reality check

| Type | Realistic range for solo dev |
|------|------------------------------|
| Small indie (Steam) | $100–$5,000 at launch |
| Mid indie (Steam) | $5,000–$50,000 at launch |
| Mobile app (premium) | $500–$5,000 MRR |
| Mobile app (F2P + IAP) | Highly variable, usually < $500 MRR unless viral |
| Hyper-casual mobile | Requires large scale, not suitable for solo |

### High-risk monetization models for small teams

- **Gacha / loot boxes**: Requires ongoing content, live ops team, player retention ops
- **Subscription with live content**: Requires constant content pipeline
- **Hyper-casual F2P**: Requires massive volume, ad revenue only works at scale

## Red Flags (Avoid These Niches)

- Genres dominated by well-funded studios (AAA shooters, sports sims)
- Games requiring large asset budgets (3D open world, licensed IP)
- Genres where the top free game already satisfies demand
- Monetization models that require live ops infrastructure

## Green Flags (Good Niches for Small Teams)

- "I can't find a game that does X" — Reddit + IH both agree
- Genre where top results are 5+ years old
- Niche with passionate but underserved community (check subreddit size vs game count)
- Mechanic that works well on mobile AND Steam (cross-platform opportunity)
- Game that can be built with procedural generation (reduces content burden)
